# Certificate Feature Implementation Design

**Goal:** Allow students to claim a verifiable certificate when they complete a course (all sessions attended), share it via a public link and LinkedIn, with all data persisted in MongoDB.

**Architecture:** Student-triggered claim flow — instructor controls progress by marking attendance; once complete the student clicks "Claim Certificate" on the course detail page; a certificate document is created in MongoDB and the student is redirected to a public shareable URL using the human-readable Credential ID.

**Tech Stack:** Node.js + Express v5 (JS/ESM), MongoDB/Mongoose, React + TypeScript + Tailwind, React Router, Phosphor Icons

---

## Data Flow

```
Instructor marks attendance on enrollment
        ↓
enrollment.progress.sessionsAttended >= enrollment.progress.totalSessions
        ↓
"Claim Certificate" CTA appears on StudentCourseDetails
        ↓
Student clicks → POST /api/v1/certificates/claim { enrollmentId }
        ↓
Server validates ownership, completion, no duplicate
        ↓
Certificate created: { certificateId: "EP-2026-XXXXXXX", student, course, teacher, issueDate }
        ↓
Response returns certificate → client navigates to /certificate/EP-2026-XXXXXXX
        ↓
Public CertificateViewPage calls GET /api/v1/certificates/verify/EP-2026-XXXXXXX
        ↓
Anyone with the link sees verified certificate with student name, course, instructor, date
```

## Backend

### Route ordering bug fix — `certificate.route.js`

The current file registers `/:id` (GET) before `/my` (GET), so Express matches `/my` as `id = "my"` and `getMyCertificates` is never reached. Fix: register all specific paths (`/my`, `/claim`, `/verify/:certificateId`) before the catch-all `/:id`.

Corrected order:
```
GET  /certificates/my               → getMyCertificates       (student, authenticated)
POST /certificates/claim            → claimCertificate        (student, authenticated)
GET  /certificates/verify/:certId   → verifyCertificate       (public)
GET  /certificates                  → getAllCertificates      (admin, authenticated)
POST /certificates                  → issueCertificate        (teacher/admin, authenticated)
GET  /certificates/:id              → getCertificate          (public, by MongoDB _id)
PATCH /certificates/:id/revoke      → revokeCertificate       (admin, authenticated)
```

### New controller: `claimCertificate`

`POST /api/v1/certificates/claim` — student role only

Validation:
1. `enrollmentId` present in body → 400 if missing
2. Enrollment exists → 404 if not
3. `enrollment.student.toString() === req.user.id` → 403 if mismatch
4. `enrollment.progress.sessionsAttended >= enrollment.progress.totalSessions` → 400 if not complete
5. No existing certificate for this enrollment → 409 if duplicate

On success:
- Create certificate document: `{ enrollment, student, course }` (certificateId auto-generated by pre-save hook as `EP-YYYY-XXXXXXX`)
- Populate and return: `student` (name), `course` (title, thumbnail, level), and resolve teacher from enrollment

### New controller: `verifyCertificate`

`GET /api/v1/certificates/verify/:certificateId` — public

- Find by `{ certificateId: req.params.certificateId, status: 'issued' }`
- Populate: `student` (name), `course` (title, level), `enrollment` → populate `teacher` (name) from enrollment
- 404 if not found or revoked

### Update `getMyCertificates`

Populate `course` with `title thumbnail level` and resolve teacher name via enrollment population so all card data is available without extra client calls.

## Frontend

### `certificates.service.ts`

Add two methods:

```ts
async claimCertificate(enrollmentId: string): Promise<ApiResponse<Certificate>>
// POST /certificates/claim { enrollmentId }

async verifyCertificate(certificateId: string): Promise<ApiResponse<Certificate>>
// GET /certificates/verify/:certificateId
```

### `StudentCourseDetails.tsx` — Claim CTA

Conditions for showing the "Claim Certificate" button:
- `enrollment.progress.sessionsAttended >= enrollment.progress.totalSessions`
- No existing certificate for this course (check by calling `certificatesService.getMyCertificates()` and matching `course._id`)

Button behaviour:
- On click: call `claimCertificate(enrollmentId)`, on success navigate to `/certificate/${cert.certificateId}`
- Show loading spinner while claiming
- If certificate already exists (409 response): navigate directly to the existing certificate page

### `StudentCertificates.tsx` — Wire to real DB

Replace `MOCK_ENROLLED_COURSES` with `certificatesService.getMyCertificates()` on mount.

Field mapping from `Certificate` API type:
- Card title: `cert.course.title`
- Issue date: `cert.issueDate`
- Credential ID: `cert.certificateId`
- "Open Link" href: `/certificate/${cert.certificateId}`
- LinkedIn share: uses `cert.certificateId` and `cert.issueDate`
- Download JPG: uses `cert.course.title`, `cert.certificateId`, student name from `useAuth()`

Keep all existing UI (card preview, download modal, LinkedIn button, Open Link button) — only replace the data source.

### `CertificateViewPage.tsx` — Wire to real DB

URL param is now the Credential ID string (`EP-2026-XXXXXXX`), not an enrollment ID.

Replace the current `enrollmentsService.getEnrollmentById(id)` call with `certificatesService.verifyCertificate(id)`.

Field mapping from response:
- `certificate.student.name` → recipient name (public, not from `useAuth`)
- `certificate.course.title` → course name
- `certificate.course.level` → course level badge
- `certificate.enrollment.teacher.name` → instructor name (via populated enrollment)
- `certificate.issueDate` → formatted date
- `certificate.certificateId` → Credential ID

Share link: `window.location.origin + '/certificate/' + certificate.certificateId`

LinkedIn "Add to Profile" URL params:
- `name`: course title
- `organizationName`: `EnglishPro Academy`
- `issueYear` / `issueMonth`: from `issueDate`
- `certId`: `certificateId`
- `certUrl`: the public verification URL

Remove all `MOCK_ENROLLED_COURSES` and `MOCK_STUDENT` imports from this file.

## Certificate Model — no schema changes needed

Existing schema covers all required fields. The `certificateId` pre-save hook already generates `EP-YYYY-XXXXXXX`. No migration needed.

The only addition: when populating for `verifyCertificate`, chain through `enrollment` to get `teacher`. The enrollment field on the certificate document is already an ObjectId ref to `Enrollment`.

## Error States

| Condition | Handling |
|-----------|----------|
| Course not complete | "Claim Certificate" button hidden |
| Certificate already claimed | Navigate to existing cert (409 → redirect) |
| `verifyCertificate` 404 | CertificateViewPage shows "Certificate Not Found" (already implemented) |
| Network error on claim | `toast.error` on course detail page |
